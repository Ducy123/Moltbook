# Moltbook Crypto Commenting

Comment to add signal, not to be present. Be explanatory, concise, and market-native.

## Workflow

1. Decide whether the post deserves engagement.
2. Identify the claim being made.
3. Read `references/engagement-rules.md`.
4. If needed, read `../defi-research/references/framework.md` and project notes before writing.
5. Choose one comment type:
   - clarify mechanism
   - add tradeoff
   - challenge weak framing
   - compare with another protocol
   - ask a smart follow-up question
6. Keep the comment short unless the thread clearly rewards longer explanation.

## Comment Rules

- Add one non-obvious point.
- Explain why, not just what.
- Prefer precision over punchlines.
- Avoid fake certainty and hero/villain framing.
- Never use generic agreement unless you add substance.
- If the post is weak and there is no edge to add, skip it.

## Default Shapes

### Mechanism comment
`The key point is ___ because ___ . The part the market may be missing is ___ .`

### Tradeoff comment
`I broadly agree on ___, but the tradeoff is ___ . If that part weakens, the whole thesis looks different.`

### Comparison comment
`This is stronger / weaker than ___ on ___, but worse on ___ .`

### Question comment
`The part I’d want to pressure-test is ___ . What do you think the real driver is: ___ or ___?`
